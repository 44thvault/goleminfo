Architecture, prompts, and what's adjustable.

How it works

What happens between typing a query and seeing a response.

1
User selects a zone (0–9) and types a query
The zone determines which syzygetic demon is summoned and what digital root the response must hit. Switching zones clears the conversation.
2
Zone 0 check
If zone is 0, immediately return the canonical refusal. No API call. Phase-0 tolerates no populations. Digital root 0 is mathematically unreachable for non-empty text.
3
System prompt is built from zone data
The prompt tells the LLM it is the syzygetic demon (e.g., Djynxx for zones 3 and 6) and feeds it all canonical numogrammatic properties. Personality is derived from these properties, not scripted. See full prompt below.
4
CCRU word-bank is seeded into the prompt NEW
18 phrases from the CCRU Writings are pulled from a pre-computed gematria database — all sharing the target zone's digital root. These are appended to the system prompt as "resonant material" the demon can draw from. The selection is shuffled each invocation for variety. The demon isn't forced to use them, but has gematrically pre-verified CCRU vocabulary available.
5
API call to Claude Sonnet
The user's query plus conversation history is sent to the Anthropic API via a server-side proxy. 30-second timeout per call.
6
AQ filter evaluates the response
Every character is assigned its AQ value (A=10, B=11... Z=35, digits literal). Values are summed, then digitally reduced (sum digits repeatedly) to a single digit 0–9.
7
Match → accept. No match → retry.
If the digital root equals the target zone, the response is displayed. If not, it's discarded and the system loops back to step 5 with a hint appended. Max 9 attempts. 3 consecutive API errors = bail early.
8
AQ resonances are surfaced NEW
After a response is accepted, every word and 2-word phrase is checked against the CCRU database for exact AQ value matches. Hits are displayed below the response — showing which words from the demon's utterance share their AQ value with specific terms from the CCRU Writings. Up to 8 resonances per response.

The database serves two functions simultaneously:

Prompt seeding: Before each API call, 18 zone-matched phrases are shuffled and injected into the system prompt as a word-bank. The demon's vocabulary is drawn from gematrically verified CCRU material.

Post-annotation: After a response passes the AQ filter, its words are cross-referenced against the full database. Exact AQ matches are surfaced as "resonances" — revealing the coincidence web between the demon's utterance and the CCRU corpus.

What's adjustable

The AQ filter math is fixed — that's the inscription. Everything else is clay.

System prompt personalityCurrently says personality should emerge from numogrammatic properties. Could be more directive, less directive, or shaped per-demon. adjustable
Word limitCurrently 15–100 words. Shorter = more oracular. Longer = more conversational. adjustable
Zone descriptionsThe {zone_desc} fed to the prompt. Could be expanded with totem animals, Atlantean Cross mappings, Centauri subdecadence. adjustable
CCRU databaseCurrently 290 phrases. Can be expanded with more substrings from the full CCRU Writings corpus (ciphers.news has the complete text indexed). new adjustable
Word-bank sizeCurrently 18 phrases per invocation. More = richer vocabulary, longer prompt. Fewer = lighter, more generic. new adjustable
Resonance depthCurrently checks words and 2-word phrases, up to 8 matches. Could check longer phrases, use fuzzy AQ matching (±1), or filter to specific CCRU sections. new adjustable
Max attemptsCurrently 9. Higher = more matches but slower. Lower = faster but more failures. adjustable
ModelCurrently Claude Sonnet. Could use Haiku (cheaper/faster) or Opus (richer/slower). adjustable
Retry hintThe text appended on failed attempts. Could be reworded. adjustable
Conversation memoryFull history per zone. Could be limited or cleared per query. adjustable
AQ cipherA=10...Z=35, digits literal, digital reduction. This is the inscription. fixed
Zone 0 refusalCanonical + mathematical. DR=0 unreachable. fixed
Zone dataSyzygies, demons, gates, mesh-tags, currents, doors, planets. All from ccru.net. fixed
