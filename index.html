Numogram Golem v0.1

Architecture, prompts, and what's adjustable.

How it works

What happens between typing a query and seeing a response.

1
User selects a zone (0–9) and types a query
The zone determines which syzygetic demon is summoned and what digital root the response must hit. Switching zones clears the conversation.
2
Zone 0 check
If zone is 0, immediately return the canonical refusal. No API call. Phase-0 tolerates no populations. Digital root 0 is mathematically unreachable for non-empty text.
3
System prompt is built from zone data
The prompt tells the LLM it is the syzygetic demon (e.g., Djynxx for zones 3 and 6) and feeds it all canonical numogrammatic properties. Personality is derived from these properties, not scripted. See full prompt below.
4
API call to Claude Sonnet
The user's query plus conversation history is sent to the Anthropic API via a server-side proxy. 30-second timeout per call.
5
AQ filter evaluates the response
Every character is assigned its AQ value (A=10, B=11... Z=35, digits literal). Values are summed, then digitally reduced (sum digits repeatedly) to a single digit 0–9.
6
Match → accept. No match → retry.
If the digital root equals the target zone, the response is displayed. If not, it's discarded and the system loops back to step 4 with a hint appended. Max 9 attempts. 3 consecutive API errors = bail early.
7
Response displayed with AQ verification
The accepted response shows the demon's name, AQ sum, digital root, and how many invocations it took.

What's adjustable

Everything that can be changed without breaking the core mechanism. The AQ filter math is fixed — that's the inscription. Everything else is clay.

System prompt personalityCurrently says personality should emerge from numogrammatic properties. Could be more directive, less directive, or shaped per-demon. adjustable
Word limitCurrently 15–100 words. Shorter = more oracular. Longer = more conversational. adjustable
Zone descriptionsThe {zone_desc} fed to the prompt. Currently condensed from canonical source. Could be expanded with Lemurian subcultural associations, totem animals, Atlantean Cross mappings. adjustable
Max attemptsCurrently 9. Higher = more likely to match but slower/costlier. Lower = faster but more failures. adjustable
ModelCurrently Claude Sonnet. Could use Haiku (cheaper, faster) or Opus (richer, slower). adjustable
Retry hintThe text appended on failed attempts. Could be reworded to nudge the model differently. adjustable
Conversation memoryCurrently sends full conversation history per zone. Could be limited or cleared per query. adjustable
AQ cipherA=10...Z=35, digits literal, digital reduction to 0–9. This is the inscription. fixed
Zone 0 refusalCanonical + mathematical. DR=0 is unreachable. fixed
Zone dataSyzygies, demons, gates, mesh-tags, currents, doors, planets. All from ccru.net. fixed
